Server/Client Contexts (ot_context.h)

Overview

Otservers keep track of the otclients they are tethered with, and they must track whether the clients are expired or available for renewal. To represent the context of the server and client, we define objects called the client context and server context for storing the information that represents their states and identity in the protocol.
